*目的：
解决当前OFFICE（WORD及WPS）在结构化长文档如投标书、需求文档等OFFICE效率低下的痛点问题，提供自动化批量处理手段，效率比纯人工提升至少100倍（估算）。
运用得力，将是各位OFFICE使用者的福音。
市场上也为空白；
其中，对WORD的API可靠高效处理做了数次试验筛选与改进，最终形成当前的代码结构；
希望有意发展此部分的专业人士能够基于此进行发展，贡献出解决痛点的优秀工具或产品；

WORD在结构化长文下的痛点问题与相应解决方法：
1. 章节序号配置复杂与应用无批量处理	-->    章节序号功能
2. 章节样式配置复杂与应用无批量处理	-->    章节样式功能
3. 题注设置设置复杂与应用无批量处理	-->    批量题注功能
4. 无批量处理不同文本对象的能力	-->    多选器
5. 文档内各文档对象跳转导航功能不易用	-->    对象导航
6. 文档中不能实现如EXCEL自动合计计算	-->    关联计算

其它改进：见《使用手册》


*进一步改进：
1. WORD2010及以上版本、WPS专业版等版本上未进行全面测试，需要有兴趣的小伙伴继续发展解决；
2. 多选器功能强大，但对长文档的处理尚需优化处理方式，否则会出现量过大造成应用无反应的情况；
3. 已奠定全文排版的技术基础，下一步可以进行全文排版功能，此功能若成，善莫大焉；
4. 与服务器端的整合与设计，还有很大空间，需有志者完善解决；
5. 不足之处，请高人多多改进；
